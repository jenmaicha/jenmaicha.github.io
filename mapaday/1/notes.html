January 1, 2019: Notes

For this map, I converted an existing map of NYC Christmas markets (and a few other points of interest) that I made for a day trip last month from Leaflet.js to Mapbox GL JS.

Approach

I based this project off a tutorial series from Mapbox (but using my own data). The basic idea is to create a data set in the Mapbox Studio dataset editor, export the dataset as a tileset, style the tileset as a layer in Mapbox Studio, add the custom style to a Mapbox GL JS map, and then add some interactivity for marker popups.

Notes

In Mapbox Style Studio: the layer gets a type. "Symbol" is a type that can be text labels or icons.

The Maki icon set is available to use. It's customizable. But the Maki icons are definitely designed and sized to blend into an overall map, rather than stand out as overlay markers on a map.

The popups work by querying all features from a specified layer/set of layers in the map style that are rendered at the point on the map where the user clicks, and then pulling info from the first element in the returned array of rendered features. Which is kind of a cool thing to be able to do... but is missing the hover action of the hand changing to a pointer when there is something clickable.

I'm not thrilled yet with the idea of all of this data being embedded into the map "style," but I can't tell if that's just because I'm used to thinking of custom data as overlay/additional to an underlying map. Hm.

Need to read up on the distinction between datasets and tilesets (see link below).
